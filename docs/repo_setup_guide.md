# Repo Setup Guide

## Option A: Use as private GitHub repo
1. Create a private GitHub repository.
2. Upload all files in this folder.
3. Keep real evidence and personal data outside the public repo.
4. Use Issues to track legal research questions.
5. Use Pull Requests to review updates to skills and templates.

## Option B: Use as local knowledge base
1. Store this folder in OneDrive / SharePoint / local secure drive.
2. Create subfolders for each case or project.
3. Copy templates into each case folder.
4. Update taxonomy and keyword dictionary over time.

## Option C: Use with AI Agent tools
1. Put `AGENTS.md` at repo root.
2. Put domain context in `CONTEXT.md`.
3. Put behavioural rules in `CLAUDE.md` or tool-specific instruction files.
4. Let the Agent read the repo before answering.
5. Require the Agent to write outputs into `/outputs`.
